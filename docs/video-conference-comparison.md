# 视频会议方案对比

## 📊 三种方案对比

| 特性 | 当前P2P方案 | LiveKit方案 | 自建SFU方案 |
|------|------------|-----------|------------|
| **支持人数** | 2-4人 | 10-50人 | 100+人 |
| **开发周期** | ✅ 已完成 | 1-2周 | 2-3月 |
| **部署难度** | 简单 | 简单 | 复杂 |
| **月成本** | ¥0 | ¥1300（自建）<br>$264（云服务） | ¥5000+ |
| **网络穿透** | ⚠️ 弱 | ✅ 强 | ✅ 强 |
| **录制功能** | ❌ 需自己实现 | ✅ 内置 | ❌ 需自己实现 |
| **维护成本** | 低 | 低 | 高 |
| **扩展性** | ❌ 差 | ✅ 好 | ✅ 很好 |

---

## 🎯 推荐选择

### **场景1：小班教学（<10人）**
**推荐**：当前P2P方案 ✅

**理由**：
- 已完成，无需额外开发
- 成本为0
- 满足小规模需求

**何时升级**：
- 当班级人数>5人时
- 出现连接问题时
- 需要录制功能时

### **场景2：中班教学（10-50人）**
**推荐**：LiveKit方案 ✅✅✅

**理由**：
- 开发周期短（1-2周）
- 性能稳定
- 成本可控（¥1300/月自建或$264/月云服务）
- 功能完善（录制、屏幕共享、白板集成）

**实施步骤**：
1. 部署LiveKit服务器（10分钟）
2. 后端集成JWT生成（30分钟）
3. 前端集成LiveKit SDK（1小时）
4. 测试验证（30分钟）

### **场景3：大班教学（>50人）**
**推荐**：自建SFU方案

**理由**：
- 支持大规模并发
- 可深度定制
- 长期成本更低

**实施步骤**：
1. 选型（Mediasoup/Jitsi）
2. 架构设计
3. 开发（2-3月）
4. 部署运维

## 🔄 迁移路径

### **阶段1：保持P2P（现在）**
```typescript
// 使用当前VideoConference组件
import { VideoConference } from './components/VideoConference';

<VideoConference
  visible={showConference}
  onClose={() => setShowConference(false)}
  roomId={roomId}
  userId={userId}
  username={username}
/>
```

### **阶段2：添加LiveKit（1-2周后）**
```typescript
// 同时保留两个组件
import { VideoConference } from './components/VideoConference';
import { LiveKitVideoConference } from './components/LiveKitVideoConference';

// 根据人数选择
{participantCount <= 5 ? (
  <VideoConference {...props} />
) : (
  <LiveKitVideoConference {...props} />
)}
```

### **阶段3：完全迁移（1-2月后）**
```typescript
// 只使用LiveKit
import { LiveKitVideoConference } from './components/LiveKitVideoConference';

<LiveKitVideoConference {...props} />
```

## 💡 功能对比

### **P2P方案限制**

**❌ 无法实现的**：
1. 录制功能（需要每个客户端录制）
2. >4人会议（网络拓扑复杂）
3. 服务器端控制（去 mute、踢人）
4. 质量监控（无法统计）
5. 选择性转发（全部转发）

**✅ 可以实现的**：
1. 基本音视频
2. 屏幕共享
3. 聊天
4. 静音/关闭摄像头

### **LiveKit方案优势**

**✅ 开箱即用**：
1. 录制功能（一键录制）
2. 人数监控（实时统计）
3. 质量控制（自适应码率）
4. 断线重连（自动重连）
5. 选择性转发（只转发活跃发言者）

**✅ 高级功能**：
1. 服务器端录制
2. 实时转录（语音转文字）
3. SIP集成（电话拨入）
4. 扩展插件（虚拟背景、滤镜）
